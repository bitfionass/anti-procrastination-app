📋 项目概况

《和拖延say拜拜》是一款针对大学生拖延行为的干预工具。基于对114名大学生的调研数据，采用"AI任务拆解 + 5分钟启动器 + 即时反馈 + 电子宠物"的组合干预方案，帮助用户克服启动困难、情绪逃避等拖延核心问题。

🚀 核心功能

🤖
AI任务拆解
输入"写论文"等模糊任务，AI自动拆解为5-8个具体步骤，降低启动心理门槛
⚡
5分钟启动器
点击"先做5分钟"，专注第一步，倒计时结束后自动鼓励，突破情绪逃避
📊
进度反馈
实时进度条、勋章解锁、成就海报，每一步都有正向反馈
🐾
电子宠物
完成任务获得食物喂养宠物，宠物心情值和成长等级随你的进步而提升

🎨 产品设计理念

针对调研发现的三大核心痛点，设计了对应的干预方案：

痛点解决方案功能
不知从何开始（25.44%）降低启动门槛AI任务拆解
情绪逃避（38.6%）最小化启动动作5分钟启动器
缺少正反馈（64.04%）即时视觉激励进度条+勋章
情感陪伴缺失（19.3%）拟人化激励电子宠物

技术选型

  • 纯前端：HTML + CSS + JavaScript，零依赖，可直接打开
  • 数据存储：LocalStorage，无需服务器，刷新不丢失
  • AI引擎：Mock模拟（可替换为真实AI API）
  • 部署方式：PWA（Progressive Web App），可安装到手机主屏幕

🛠️ 开发过程

项目从方案到成品，经历了系统化的开发流程，总耗时约2.5小时。

阶段1 · 10分钟
方案评审
对207行原始方案进行系统性评审，发现5个问题：数据矛盾、占比缺失、人员不匹配、优先级不明确、缺竞品分析
阶段2 · 15分钟
功能映射
将6个功能模块精简为4个MVP核心模块，确定技术选型：原生JS + LocalStorage + Mock AI
阶段3 · 30分钟
代码开发
逐模块编写前端代码：HTML结构 → CSS样式 → Mock AI引擎 → 电子宠物 → 主逻辑，共55KB代码
阶段4 · 5分钟
初版测试
启动本地HTTP服务器，验证页面加载和基础交互
阶段5 · 20分钟
视觉重设计
全面重构设计系统：CSS变量、渐变背景、毛玻璃效果、微交互动画，CSS从12KB扩展到17KB
阶段6 · 15分钟
功能测试
使用Playwright自动化测试，13/13测试用例全部通过
阶段7 · 10分钟
代码审查
检查DOM引用、CSS一致性、事件绑定、错误处理，无阻断性问题
阶段8 · 15分钟
PWA升级
添加Service Worker、manifest.json、App图标，支持安装到手机主屏幕
阶段9 · 20分钟
微信小程序版
将核心功能移植为微信小程序，11个文件，可在微信开发者工具直接运行
阶段10 · 10分钟
教学演示PPT
生成21页教学演示PPT，展示"从方案到App"的全流程
阶段11 · 5分钟
打包部署
上传GitHub，部署到GitHub Pages，生成访问链接和二维码
